# zutomot.com 事業計画書

## Astro中心JAMstackによる中小・個人ストア向けWebテンプレート事業

---

## 1. 事業概要

### 1.1 目的

中小・個人ストア向けに
**「速い・安い・壊れにくい」公式Webサイト**を
**テンプレ固定・自動化前提**で提供し、
Googleマップ経由の集客機会損失を解消する。

将来的には

* WordPress運用の負担軽減
* セキュリティ・保守コスト削減
  を訴求し、**WPからの段階的な乗り換え**を促進する。

---

### 1.2 ターゲット

* Googleマップに店舗情報はあるが、

  * 公式サイトがない
  * SNSリンクのみ
  * 表示が遅い／スマホ非対応
* 小規模店舗（飲食・美容・整体・スクール等）
* IT運用にリソースを割けない事業者

---

## 2. ビジネス方針（最重要原則）

### 2.1 開発・運用の基本方針

* **テンプレート固定**
* **要件の個別対応は行わない**
* **オプションは事前定義した範囲のみ**
* **開発側の作業はすべて自動化・再現可能にする**

> 「案件」ではなく「プロダクト」として提供する

---

### 2.2 やらないこと（明文化）

* 広告運用代行
* 個別KPI設計・ABテスト
* イベント定義のカスタマイズ
* クライアントごとのタグ設計
* WordPressプラグイン互換再現

---

## 3. 提供内容（プロダクト設計）

### 3.1 サイトテンプレート構成（固定）

#### 基本テンプレ（LP型 / Small Site型）

* Home（ヒーロー・特徴）
* メニュー / サービス
* 料金
* アクセス（Googleマップ埋め込み）
* お問い合わせ（フォーム）
* プライバシーポリシー

※ ページ構成・導線は全クライアント共通

---

### 3.2 技術スタック（固定）

* フレームワーク：Astro（SSG前提）
* スタイリング：Tailwind CSS
* ホスティング：Netlify / Cloudflare Pages 等
* フォーム：外部サービス（サンクスページ遷移方式）
* CMS（必要時のみ）：microCMS 等

---

## 4. GTMオプションの位置づけ

### 4.1 方針

**GTMは「計測の土台」まで**
広告・マーケ施策には踏み込まない。

---

### 4.2 GTMオプションに含める内容（固定）

#### 実装範囲

* GTMコンテナ埋め込み

  * Astroテンプレートに標準組み込み
  * Container IDは環境変数 or CMS設定で注入
* GA4 基本計測

  * page_view
* 固定イベント計測（テンプレ共通）

  * 電話リンククリック
  * LINE / 予約リンククリック
  * フォーム送信（サンクスページ遷移）

#### イベント仕様

* イベント名・パラメータは固定
* 変更・追加は不可（要件外）

---

### 4.3 明確に「含めない」内容

* Google広告 / Meta広告の設定
* コンバージョン定義の調整
* タグマネージャー内の運用代行
* Cookie同意設計の個別対応
* 計測精度・成果保証

> 計測は **ベストエフォート** とし、
> 広告成果・数値改善の責任は負わない。

---

## 5. オプション設計（例）

| オプション       | 内容                 | 工数  |
| ----------- | ------------------ | --- |
| GTM埋め込み     | GTM + GA4 + 固定イベント | 低   |
| CMS導入       | News更新等            | 中   |
| WordPress移行 | 重要ページのみ            | 中〜高 |
| 多言語         | 英語1言語のみ            | 中   |
| 写真差し替え      | 支給素材のみ             | 低   |

※ オプションはすべて事前定義制
※ 未定義オプションは受けない

---

## 6. 料金モデル（例）

### 初期費用

* テンプレ導入費
* GTMオプション費（有償）

### 月額費用

* ホスティング
* 軽微修正（回数・時間制限あり）
* 監視・保守
* （GTM利用時）計測基盤維持

> 初期＋月額の**サブスクモデル**を基本とする

---

## 7. 営業・獲得戦略

### 7.1 アプローチ方法

* Googleマップ上で
  「公式サイトが弱い / 無い店舗」を抽出
* 無料診断（30分・テンプレ化）

  * 地図→サイト導線の欠落
  * 表示速度・スマホ適正
  * 競合との差分

---

### 7.2 セールスポイント（固定トーク）

* 表示が速い
* 更新・管理が不要
* セキュリティ対応不要
* 広告・SNSと“繋げられる土台”ができる

---

## 8. スケジュール（初期90日）

1. テンプレート1種完成（LP型）
2. デモサイト3業種作成
3. 提案資料・価格表整備
4. 営業リスト100件作成
5. 初期導入3件獲得
6. 運用フロー微調整 → 横展開

---

## 9. リスクと対策

| リスク    | 対策           |
| ------ | ------------ |
| 要望肥大化  | テンプレ固定・契約明文化 |
| 計測トラブル | ベストエフォート明記   |
| 広告成果責任 | 広告非対応を明確化    |
| 工数増加   | オプション厳格管理    |

---

## 10. まとめ

本事業は
**「作らない勇気」「やらない線引き」**によって成立する。

* Astro × JAMstackで開発コストを最小化
* テンプレ固定で運用を自動化
* GTMは“未来の拡張余地”として最低限のみ提供

これにより、
**少人数・低工数でもスケール可能なWeb提供モデル**を構築する。

# 実装詳細計画：Astroテンプレート事業

## 1. 技術基盤の詳細定義
### ディレクトリ構成 (src/)
- `components/`
    - `ui/`: ボタン、バッジなどの最小単位
    - `sections/`: Hero, Features, Access などのセクション単位
    - `common/`: Header, Footer, GTM
- `layouts/`: BaseLayout.astro
- `pages/`: index.astro (LP用), [slug].astro (動的ページ)
- `styles/`: global.css (Tailwind)
- `types/`: index.ts (共通の型定義)

### Tailwind CSS 設定
- `tailwind.config.mjs` にて `theme.extend.colors` に `brand-primary`, `brand-secondary` を定義
- CSS変数 (`--color-primary` 等) を介して、configを変更せずに色を一括変更可能にする

## 2. 共通コンポーネントのProps/型定義
- すべてのセクションコンポーネントは `interface Props` を持ち、TypeScriptで厳密に管理する
- 例: `Hero.astro`
    - title: string
    - description: string
    - backgroundImage: string
    - ctaLabel: string
    - ctaLink: string

## 3. GTMコンテナの標準レシピ (.json)
- コンテナIDを環境変数 `PUBLIC_GTM_ID` で受け取る
- デフォルトで以下のタグを設定したJSONファイルを用意
    - GA4 設定タグ
    - 電話リンククリックイベント (data-gtm-event="tel")
    - LINEリンククリックイベント (data-gtm-event="line")
    - フォーム送信完了 (サンクスページ到達)

## 4. microCMSスキーマ定義
- `news`: タイトル、公開日、カテゴリ、本文、サムネイル
- `pages`: スラッグ、メタタイトル、メタディスクリプション、本文 (リッチエディタ)

## 5. フォーム外部サービスの連携仕様
- 第一候補: **SSForm** (国産・設定が容易)
- 実装方法:
    - `form` タグの `action` に指定URLを設定
    - `on submit` でGA4イベントを発火させた後、指定のサンクスページへ遷移

## 6. デプロイ/リポジトリ管理フロー
- GitHub Templateリポジトリを使用
- 案件ごとに `use this template` で新規リポジトリ作成
- Netlify/Cloudflare Pages と自動連携 (mainブランチ)

## 7. デモサイト3業種
1.  **カフェ/飲食店**: メニュー重視、落ち着いた配色
2.  **整体院/クリニック**: 信頼感、予約ボタンの強調
3.  **パーソナルジム/スクール**: 活力、特徴(ベネフィット)の訴求

## 8. 次のステップ
- [ ] `src/` 配下の基本フォルダ作成
- [ ] `BaseLayout.astro` の実装
- [ ] Tailwind CSS の変数定義

## 0. 前提（業種特性の整理）

リラクゼーションサロンは以下の特性を持つ：

* 医療・治療ではない（※表現規制が重要）
* Googleマップ・地域検索依存
* サービス内容は似通っている
* 更新頻度は低い
* 「安心感」「落ち着き」が重要

👉
**誇張・効果保証・治療表現を物理的に出せない設計**が必須。

---

## 1. 文言設計の基本ルール（業種特化）

### 1.1 文体・トーン

* 丁寧語（です・ます）
* 柔らかい表現
* 効果・改善を断定しない

### 1.2 業種特有の禁止表現（自動フィルタ対象）

* 治る／改善する／効く
* 医療／治療／診断
* 肩こり解消・腰痛改善（※断定不可）
* 国家資格・医師連携（虚偽防止）

---

## 2. ページ別：変数定義仕様

# Design仕様書（Webインフラ提供事業・初期版）

## 1. 初期テンプレ種別決定

### 仕様

* **初期テンプレは1業種・1テンプレのみ**
* 対象業種：**リラクゼーションサロン（個人・小規模）**
* デザイン方針：

  * シンプル
  * 情報整理優先
  * 写真1〜3枚でも成立

### 理由

* 自動化・検証・例外排除を最優先
* 複数テンプレは「運用負債」になりやすいため初期は避ける

---

## 2. 複数テンプレ展開方針整理

### 方針

* **横展開は「業種別」ではなく「構造別」**
* デザイン差分ではなく **データ差分**で吸収

### 展開ルール

* テンプレ追加条件：

  * 20社以上の実績
  * 構成差分が明確
  * 既存テンプレで吸収不可

### 禁止事項

* 顧客ごとのテンプレ分岐
* 色・配置の自由指定
* デザインAB系の派生

---

## 3. 必須ページ洗い出し
